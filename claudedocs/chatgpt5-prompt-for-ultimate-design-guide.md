# ChatGPT-5 Pro 用プロンプト: 万能デザイン指示書作成

このプロンプトをChatGPT-5 Pro Thinkingにコピペして、万能「AI感排除・人の手デザイン」指示書の材料を集めてください。

---

## 🎨 **プロンプト本文**

```
あなたは一流UIデザイナーとして、以下の質問に答えてください。
目的: AI生成UIから「人の手」を感じさせるデザインへの変換指南書を作成

---

【Phase 1: デザインの意思決定プロセス分析】

一流のUIデザイナーが「余白」「配色」「フォント」を決める時、
どんな順序で、どんな根拠で判断しているのか？

特に知りたいこと:
- 余白を決める時の「感覚」の正体（視覚的リズム？認知負荷？）
- 色を選ぶ時の「直感」の背後にある理論
- フォントサイズを微調整する時の判断基準（0.5px刻みで調整する理由）
- 「ここはこうじゃない感」を感じる瞬間の共通パターン

AI生成と人間デザインの違いを、
「プロセス」「判断基準」「微調整の理由」の3軸で分析してください。

---

【Phase 2: AI臭さの正体を特定】

以下のようなAI生成UIに共通する「AI臭さ」の正体を分析してください:

観察事例:
- すべての要素が「完璧に整列」している（人間は意図的に崩す）
- 余白が「数学的に均等」（人間は視覚的重要度で調整）
- すべてのボタンが「同じ角丸」（人間は文脈で変える）
- グラデーションが「綺麗すぎる」（人間は微妙なノイズを入れる）
- アニメーションが「滑らかすぎる」（人間は意図的にカクつかせる）

質問:
1. なぜ人間デザイナーは「不完全さ」を意図的に残すのか？
2. 「整いすぎている」ことがなぜ違和感を生むのか？
3. 一流デザイナーが「崩す」時の判断基準は？
4. モダンUI（Stripe、Linear、Notion系）特有の「人間らしさ」とは？

---

【Phase 3: 一流と二流の境界線】

以下の2つのデザインを比較した時、何が「一流」を分けるのか？

比較例:
A. 完璧に整列、均等な余白、滑らかなアニメーション
B. 微妙なズレ、意図的な不均等、カクついた演出

質問:
1. 「一流」の定義を5つの具体的指標で示してください
2. なぜApple/Airbnb/Stripeは「一流」と呼ばれるのか？（デザインシステムの観点）
3. モダンUIにおける「一流」とは？
4. 意図的な「不完全さ」と「雑さ」の境界線
5. 「洗練」と「無機質」の違い

実例を挙げて、視覚的・技術的に分析してください。

---

【Phase 4: フレームワーク制約下での人間らしさ】

Chakra UI v3のようなデザインシステムを使いながら、
どうやって「人の手」を感じさせるUIを作るか？

技術的制約:
- トークンベースのデザイン（spacing.4, colors.blue.500）
- コンポーネントの再利用性重視
- 型安全性とデザインの柔軟性のトレードオフ

質問:
1. デザインシステムを「崩す」技術的手法（CSS-in-JSでの微調整）
2. トークンを使いながら「意図的なズレ」を作る方法
3. アニメーション easingの「人間らしい」カスタマイズ
4. レスポンシブデザインでの「手作り感」の保ち方
5. Next.js + Chakra UIでの実装時の注意点

コード例を含めて具体的に示してください。

---

【Phase 5: 実践的チェックリスト】

以下のような実装時のチェックリストを作ってください:

目的: コンポーネント実装時に「AI臭さ」を検出・修正

チェック項目例:
□ すべての余白が4の倍数になっていないか？（人間は3px, 5px, 7pxも使う）
□ すべてのボタンが同じ角丸になっていないか？
□ アニメーション easingがease-in-outだけになっていないか？
□ グラデーションが数学的に均等になっていないか？
□ すべてのフォントサイズが偶数になっていないか？

質問:
1. 上記以外に50個のチェック項目を作成してください
2. 各項目に「なぜNG」「どう直す」を付けてください
3. 優先度（高/中/低）を設定してください
4. モダンUI特有のチェック項目を追加してください
5. Chakra UI v3での実装時の具体的修正方法を示してください

---

【Phase 6: Before/Afterの実例分析】

以下のような実装を「AI生成」→「人の手」に変換してください:

Before（AI生成っぽい）:
<Box
  p={4}
  bg="blue.500"
  borderRadius="md"
  boxShadow="lg"
>
  <Text fontSize="lg">ボタン</Text>
</Box>

After（人の手っぽい）:
<Box
  p="17px"  // 意図的に中途半端な値
  bg="#4169E1"  // トークンじゃなく直指定
  borderRadius="3px"  // 微妙に角ばってる
  boxShadow="2px 3px 0px rgba(0,0,0,0.3)"  // 立体的な影
  transform="translate(0.5px, -0.5px)"  // 微妙なズレ
>
  <Text fontSize="18px" letterSpacing="0.03em">ボタン</Text>
</Box>

質問:
1. 上記の変換理由を各プロパティごとに説明
2. 20個のBefore/After実例を作成（ボタン、カード、モーダル、フォーム、ナビゲーションなど）
3. なぜこの変更が「人間らしさ」を生むのか理論的に説明
4. モダンUI、ミニマルUIの2パターンで示す
```
