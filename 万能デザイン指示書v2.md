1. Art Direction（世界観・色・質感）
原則（なぜそうするのか）

テーマは「夜の城で行われる紋章の儀式」

夜空・満月・焚き火・ドラゴン・騎士・魔法・紋章・古い紙・石畳を
「カードを読み解く儀式」に結びつけて、一つの文脈に統合する。

HD-2D の構造を UI にも適用する

2Dの絵（背景・カード・アイコン）を、立体的な光と影で包む設計。

UIパネルを「ページの上のHTML要素」ではなく、
世界の中に存在する石板や操作盤として扱う。

v1 の“微差”を世界観レベルでも適用する

石の欠け・金属のくすみ・紙の黄ばみ・光のゆらぎを、
均一なテクスチャや完璧なグラデではなく、僅かなムラとして扱う。

具体例

ビジュアルの芯

背景：

深い紺の空（星少なめ）＋雲に隠れかけた満月。

城壁や塔、石畳が月光と焚き火で部分的に照らされる。

前景オブジェクト：

焚き火 / 燭台 / 魔法陣からの光が、床に不均一なハイライトを落とす。

UIパネル：

石板…暗い石のベースに、細い黄金の装飾ライン。

古文書…エッジが少し破れた紙、インク染みのようなノイズ。

金属…鈍い真鍮色、完璧な鏡面ではなく擦り傷・くすみあり。

HD-2D から借りるポイント

前景（UI、カード）と奥（背景）のコントラスト差で奥行きを出す。

画面の四隅を暗く落とすヴィネットで視線を中央へ集約。

光源（満月・焚き火・魔法陣）の位置に応じて、UI上部/下部のハイライト方向を揃える。

安易に真似すると安っぽくなるポイント

画面全体に強いブルームをかけて情報が白く飛ぶ。

「とりあえず全部光らせる」方向で、重要度が分からなくなる。

ドット絵を無理に UI 部品にまで適用し、文字や数字が潰れる。

NG例

ベタ塗りの紺背景＋真っ黒なUIパネルで、石や紙の質感がない。

すべてのハイライトが白一色で、光源の種類（炎／月光／魔法）が区別されていない。

UIパネルだけモダンWebっぽいフラットデザイン（淡い青グラデ＋白いカード）。

「夜」なのに、全体的にコントラストが高すぎて眩しい。

チェックリスト

 1画面見ただけで、「夜の城で紋章を扱う儀式」がテーマだと分かるか。

 石／金属／紙／炎／月光／魔法といったモチーフが、UIにもどこかで現れているか。

 HD-2D らしい「前景と背景の距離感」が、UIレイヤーにも反映されているか。

 光源の種類と方向が、大きく矛盾していないか。

2. Layout & Components（UI構造と部品）
原則

レイヤーで考える

背景：城・空・焚き火・環境

シーンパネル：フェーズごとの大枠（ロビー／配布中／推理中／結果）

コンポーネント：カード・ボタン・HUD・ログ・ダイアログ

演出レイヤー：カード飛翔、魔法エフェクト、月光のブラーなど

v1 の「整いすぎ禁止」をゲームUI用に拡張

角丸・影・余白・グリッドは、コンポーネントごとに微差を持たせる。

「カード」「結果パネル」「設定モーダル」など用途で個性を付ける。

カードゲームとしての情報優先順位を明確にする

手札 / 場札 / プレイヤー情報 / フェーズ状態 を常に最優先レイヤーに。

具体例

主要コンポーネントの運用ルール（目安）

コンポーネント	余白	角丸	影	枠線	ノイズ/テクスチャ
カード	内側: 幅の 10〜14% / 外側: 6〜10px	4〜6px	下向きにやや強め、2層	暗い金属＋内側ハイライト1px	紙ノイズ＋ごく薄い汚れ
プライマリボタン	横: 16〜20px / 縦: 10〜14px	6〜8px	下に短く強め	黄金系2重ライン	わずかなブラシストローク
HUDバー	縦: 6〜10px / 横方向ゆとり広め	3〜4px	浅く短く	暗色 1px	金属ノイズ
ダイアログ	内側余白広め（上下>左右）	8〜10px	柔らかく広め	装飾的フレーム	紙ノイズ＋縁に暗がり

HTML/CSS感を消すための「崩し方」（v1 をゲーム向けに翻訳）

余白を 4 の倍数だけで決めず、カードやパネルで 19px / 21px / 23px などの僅差を許容。

シャドウを一種類に統一せず、重要度と面積に応じて 2〜3種使い分ける。

罫線や枠線を完全な #E2E8F0 一本線ではなく、濃淡・太さ・破線を使い分ける。

テクスチャ画像を bgImage: "url(/stone-noise.png), linear-gradient(...)" のように重ねる。

フェーズごとの UI 密度・コントラスト

ゲーム開始 / ロビー

背景：城と空をよく見せる（暗すぎない）。

UI：要素数少なめ、カード無し。

目的：世界観に没入させつつ、「これから始まる」期待感。

カード配布中

背景：わずかに暗転、焚き火の光を強調。

UI：テキスト少なめ、「カードを配布しています…」＋簡単な説明のみ。

目的：カードのモーションに集中させる。

推理・相談中

背景：暗めに落として、手札・場札を最前面に。

UI：カード・プレイヤー名・発言ログなど情報多め。

目的：カード情報と会話に集中。

結果表示

背景：成功なら明るい焚き火、失敗なら一瞬暗転＋火の粉。

UI：大きな結果パネル＋スコア＋簡潔なフィードバック。

目的：「やりきった感」「次もやりたくなる余韻」。

NG例

すべてのコンポーネントの角丸が 8px 固定。

boxShadow="md" を全カード・ダイアログ・ポップオーバーに使い回している。

ロビー画面と結果画面の密度／コントラストがほぼ同じで、ドラマがない。

カード、ボタン、HUD が「普通の Web ダッシュボード」のコンポーネントと見分けがつかない。

チェックリスト

 カード／ボタン／ダイアログ／HUD／ローディング／結果画面で、余白・角丸・影・テクスチャのルールが定義されているか。

 v1 の「4の倍数だけ」「角丸全部同じ」「影全部同じ」といった AI臭さが出ていないか。

 ロビー／配布中／推理／結果で、明度・密度・演出のレベルがちゃんと切り替わっているか。

3. Color & Lighting（色とライティング）
原則

パレットは「夜」「焚き火」「月光」「魔法」「金属」「紙」を軸に設計。

v1 の「面積効果」「コントラスト運用」を踏まえ、
面積が広い色ほど彩度/明度を抑える。

光源は最低 2 種類（満月＋焚き火）を前提とし、UI側にも反映する。

具体例

ベースパレット（イメージ）

夜空・城：

紺〜黒：#070c1a ～ #111827

石畳・城壁：

暗いグレー〜茶：#2a2520 ～ #3a3128

紙・革：

深いブラウン：#2b2119 ～ #3b2a20

金属：

鈍い金：#d4a947 ～ #f0c76b（UIではハイライトとして小面積で使用）

状態色（成功／失敗／警告／情報）の世界観対応

成功（Success）

焚き火に照らされた黄金＋やや落ち着いた緑。

例：bg="#2a3a24", borderColor="#8fbf5a", テキストは淡い黄緑。

失敗（Error）

火が消えかけた赤、焦げた炭の暗さ。

例：bg="#2b1515", borderColor="#a43c32", エフェクトに火の粉。

警告（Warning）

焚き火の強いオレンジ。

例：bg="#3a2712", borderColor="#e09a3a".

情報（Info）

月光の青白。

例：bg="#151b2b", borderColor="#7fa5ff".

複数光源下での UI 表現

満月ライト：

UI上部または左上に薄い青白ハイライト。

焚き火ライト：

UI下部または右下にオレンジのボトムグロー。

魔法ライト：

特定フェーズ（必殺技、判定時など）のみ青／紫の一時的な光として使用。

NG例

成功＝明るい黄緑 (#00FF00)、失敗＝眩しい赤 (#FF0000) など、
Web UI 的な原色ステートカラーをそのまま使う。

UIパネルが純黒〜純白グラデで、背景世界から完全に浮いている。

画面全体が常に青く、焚き火の暖色や紙のブラウンがまったく感じられない。

チェックリスト

 背景・UI・光の色が、「夜／城／焚き火／月光／魔法／金属／紙」と矛盾していないか。

 成功／失敗／警告／情報の色は、世界観を壊さず、状態も一目で分かる設計か。

 UIに当たる光の向き・色が、シーン全体の光源と大きく矛盾していないか。

4. Typography（タイポグラフィ）
原則

「レトロRPG感 × 現代可読性」のハイブリッド。

v1 の「x-height・行間・字間を微差で調整する」考え方をそのまま適用。

情報の役割（タイトル／本文／数字／ボタン）ごとにフォントキャラを分ける。

具体例

用途ごとのキャラクター

タイトル／フェーズ名：

少し装飾のあるセリフ、もしくはレトロ感のあるディスプレイ系。

字間はわずかに広め（+0.02em）、行間は低めで重心を下げる。

本文：

現代的で読みやすいゴシック／サンセリフ。

行間は 1.5〜1.7、最大行幅は 60〜70 字程度。

数字（スコア・SP・ラウンド数）：

等幅 or 等幅風の数字。

文字サイズは本文＋2〜4px、太さは Bold。

ボタンラベル：

語尾や文体は世界観に合わせる（例：「開始する」「儀式を始める」）。

タイトルよりは軽く、本文よりは太く。

数字をゲームらしく“気持ちよく”見せる

HP/SP：

残量に応じて色をじわっと変化（フル→黄色→赤）させる。

減少時に数値が少し揺れてから落ち着くアニメーション。

スコア：

獲得時に数字が一瞬だけ拡大→縮小（1.05→1.0）。

黄金の細いアウトライン＋わずかなグロー。

NG例

全てを同じゴシック／同じサイズ（16px）で表示する。

タイトルが Webサイトの見出しのような「英語・オールキャップ」で、RPG感が薄れる。

数字が細い書体で夜背景に埋もれている。

チェックリスト

 タイトル／本文／数字／ボタンでフォントキャラやサイズが明確に分かれているか。

 v1 の方針どおり、行間／字間／サイズが“均一すぎる”状態になっていないか。

 数字情報が暗めの背景でも即座に読み取れるか。

5. Iconography & Symbol（アイコン・記号）
原則

アイコンは「情報記号」であると同時に、「紋章」の一部として世界観を担う。

v1 の「線幅の均一さ」を避ける考え方を、ゲームアイコンにも適用する。

SP・必殺技・状態異常・バフ/デバフは説明無しで分かるレベルを目指す。

具体例

SP・必殺技

SPゲージ：

月の位相（新月→半月→満月）で SP 量を表現。

満タンで満月アイコンが一度だけ柔らかく点滅。

必殺技アイコン：

ドラゴン、剣、紋章の組合せ。

発動可能時のみ内側から黄金に発光。

状態異常・バフ/デバフ

毒：緑の霧＋ドクロ or 牙のシルエット。

麻痺：雷＋ヒビ割れエフェクト。

バフ：上向き矢印＋光の羽／紋章の輝き。

デバフ：下向き矢印＋枯れた蔦 or ひび割れ。

線・塗り・ハイライト

線幅：

16px アイコンで 2〜3px 程度、アウトラインは太め。

角：

完全な直角ではなく、0.5〜1px 単位で丸みや欠けをつける。

塗り：

ベース色＋内側に少し暗い影＋上辺に薄いハイライト。

NG例

無料アイコン素材サイトのフラットアイコンをそのまま貼っている。

線幅 1px の繊細すぎる線で、16px 表示時に潰れている。

UI全体の色ルールと無関係な、カラフルすぎるアイコン。

チェックリスト

 SP／必殺技／状態異常／バフ／デバフは、文字を読まなくても識別できるか。

 線幅・角丸・ハイライトの付け方に、一貫した「序の紋章IIIらしさ」があるか。

 v1 の「線幅が全部同じ」AI臭さに陥っていないか。

6. Motion & Animation（モーション・アニメーション）
原則

HD-2Dらしい「重さ」「タメ」「わずかなぎこちなさ」を大事にする。

v1 の「アニメが滑らかすぎ問題」を避けるため、
イージングとタイミングに“人の手癖”を入れる。

フェーズ（待機→配布→せーの！→結果）ごとにモーションの役割を規定する。

具体例

カード配布

GSAP / Pixi.js で

出発時：少し遅い → 中盤加速 → 着地直前に減速 (cubic-bezier(.2,1,.3,1) 系)。

1枚ごとに 60〜90ms のディレイ、軌道に 1〜2px のランダム揺れ。

着地時に 1〜2px のオーバーシュート→戻り。

ドラッグ＆ドロップ

ドラッグ中：

カードが 2〜3度だけ傾く、影がマウスの下方向に追従。

ドロップ時：

100〜150ms のタメ → 成功/失敗エフェクト。

フリップ（表裏切り替え）

Y軸回転 0→90度で一瞬停止（中身切替）→90→180度。

中央付近で一度だけ光の筋が走る。

満月スピナー

回転速度は一定ではなく、

前半やや速め → 後半少し遅く → 最後に小さく加速 のループ。

星や魔法の粒子が不規則なタイミングで点滅（v1 の flicker 的な不均一）。

NG例

すべてのアニメーションが ease-in-out 300ms 固定。

スピナーがブラウザのデフォルト SVG ローディングと見分けがつかない。

カードの移動が線形＋ランダムなしで、「スライドショー」感が強い。

チェックリスト

 カードの配布・フリップ・ドラッグに、「重さ」「タメ」「微妙な揺らぎ」が入っているか。

 ローディングやフェーズ転換で、プレイヤーの気持ちをちゃんとケアできているか。

 v1 の「アニメが滑らかすぎ」AI臭さにはまっていないか。

7. “AI感”を避けるルール（v1 ゲーム版アップデート）
原則

v1 の 「AI臭さの12パターン」 をゲームUIにマッピングし直す。

特に以下のゲームUI特有の要素を重点監視対象とする：

ステータスバー／ゲージ

カードレイアウト

ローディングスピナー

エフェクトパーティクル

結果画面の演出

具体例：v1 の 12 パターン → 序の紋章III の NG 例

すべて完璧に整列

カードがグリッド上にぴったり均等で、「儀式の場」ではなく「ダッシュボード」に見える。

余白が数学的に均等

プレイヤー名・カード・ログの余白が全部 16px できっちり揃い、粗密がない。

角丸が全部同じ

カード／ボタン／モーダル／HUD が全て 8px で、役割差が消える。

グラデが綺麗すぎ

HPバーが Photoshop のデフォルトっぽいツルツルグラデで、質感ゼロ。

アニメが滑らかすぎ

カード配布／スピナー／エフェクトが、全て一定速度でぬるぬる動く。

シャドウが均質

カードもモーダルもボタンも、同じ md シャドウで奥行きの差がない。

フォントサイズが偶数だけ

14 / 16 / 18 / 20px しかなく、“手で調整した感じ”がない。

間（空白）が呼吸していない

ログや説明文の行間が全部同じで、緊張と緩和のメリハリがない。

コントラスト過剰／不足

夜の背景上で UI が真っ白に近く、眩しすぎる or 逆に全部暗く潰れている。

罫線や境界が機械的

ボードの区切り線が、全部 #E2E8F0 の 1px 実線。

アイコン線幅が一定

SP／状態異常／メニューアイコンなどすべて同じ太さ・スタイルで、世界観が希薄。

ステート変化がテンプレ通り

ホバー＝色が少し明るくなるだけ、押下＝少し暗くなるだけ、といったUIキットのデフォルト。

ゲームUI特有の追加 NG パターン

ステータスバー・ゲージが機械的な均一グラデ

例：HPバーが CSS の linear-gradient 一発で、ノイズもムラもない。

ローディングスピナーがテンプレ SVG 感

汎用の円形インジケータが回っているだけで、世界観ゼロ。

パーティクルが完全等間隔・等サイズ

焚き火の火の粉が、コピー＆ペーストしたような配置で飛んでいる。

カード裏の模様が規則的タイル

紋章パターンがきれいにタイル状に並び過ぎて、「布」ではなく「壁紙」感。

「わざと崩す」ための典型パターン（ゲームUI版）

ゲージのムラ

HP/SPバーに、ノイズテクスチャを重ねて微妙な明暗ムラを入れる。

端のハイライト幅を左右で少し変える。

ローディングのゆらぎ

満月スピナーの回転速度を 1 周の中で変化させる。

星の点滅タイミングを不規則に（v1 の flicker キーフレーム再利用）。

カードの“手置き感”

手札のカード位置や角度に ±1〜2px / ±1〜2度のランダムを入れる（ただし毎フレーム変えない）。

ここでは「考え方の例」として、実装寄りのサンプルのみ提示。
実プロジェクトではテーマトークン化・コンポーネント化前提で調整する。

原則

v1 の「微差を許容するテーマ設計」を踏まえ、
HD-2D の世界観に合わせた 色・影・半径・アニメーション をトークン化する。

すべてをトークンに閉じ込めるのではなく、局所的な“崩し”には直指定も許容する。

実装全体のチェックリスト

 v1 で定義したフォントサイズ／角丸／影／トランジションの方針を、
序の紋章III 用に上書き拡張したテーマがあるか。

 「プライマリボタン」「重要フェーズパネル」「カード」など、
世界観を代表するコンポーネントには 直指定の“手癖” を許容しているか。

 Pixi.js / GSAP で行う演出と、Chakra UI 側のアニメーションがぶつかっていないか。

8-1. HD-2D風ローディングスピナー（満月）

※ここでは「満月スピナー」の方針のみを示し、具体的なコードはあえて記載しない。

イメージ

- 中心にわずかにムラのある満月ディスク（完全なフラット円ではなく、色ムラ・陰影あり）。
- 外周にごく薄い光の輪と、さらにその外側に淡いリング枠。
- 回転は一定速度ではなく、0%→30%→60%→100% で角度の進み方を変えるなど、わずかに不均一。
- 星や粒子は 3〜4 個程度を不等間隔に配置し、点滅タイミングもランダム気味に揺らす。

ポイント

- 一定速度回転を避け、角度の進み方やイージングに“手癖”を入れる。
- v1 の「滑らかすぎるアニメ」を避けるため、ベジェカーブ＋キー位置をずらして「ほんの少しぎこちない」動きを作る。
- 大きく光らせすぎず、周囲UIの邪魔にならない「儀式の合図」レベルに留める。

8-2. 重要ボタン（ゲーム開始ボタン）

※ここも具体コードではなく、スタイル方針のみを記述する。

イメージ

- 石板／金属パネルの上に載っているような、重量感のある長方形ボタン。
- 背景は上が暗く、下に向かって少し明るくなる縦グラデーション（黄金〜真鍮系）。
- テキストはやや広めの字間＋強めのテキストシャドウで「刻印」感を出す。
- 外枠1pxの濃い枠線＋わずかなインナーシャドウで、物理的な厚みを感じさせる。
- 数秒に一度だけ、柔らかいグロー（光のにじみ）がふわっと出て戻るパルスアニメ。

ポイント

- v1 で示した「中途半端な数値」「押し込みの物理感」を HD-2D 世界観の色・影に置き換える。
- 常時ギラギラではなく、「押してほしいけど落ち着いている」程度のアテンションに抑える。
- ホバー／アクティブ時の上下移動量や影の変化量は、2〜3px 程度の“人間的な”誤差感を意識する。

8-3. 「カードを配布しています…」ローディングパネル

※このセクションもコードではなく、演出とUI構造の方針のみを残す。

イメージ

- 画面中央やや下に、小さな儀式用パネルが浮かぶイメージ。
- 背景は少しだけ暗転＋軽いブラーで、今のフェーズが「切り替え中」であることを示す。
- パネル内で 2〜3枚のカードシルエットが、僅かに時間差をつけてふわっと浮き沈みする。
- 見出しテキスト：「カードを配布しています…」は黄金寄りのカラー＋軽いテキストシャドウ。
- 補足テキストとして、世界観にあった一文（例：「月の加護により、手札が整うまで少しお待ちください」）を小さく添える。

ポイント

- v1 の「余白微差」「影の多層」「y軸の僅差アニメ」をゲームUIに転写するイメージで設計する。
- 卡ードの動きは 1〜2枚ずつに留め、情報量を増やしすぎない（フェーズの分かりやすさを優先）。
- 「ローディング中の不安」を和らげる、安心感のある演出に寄せる（激しすぎるエフェクトは避ける）。
