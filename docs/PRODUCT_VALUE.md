# 序の紋章 III（オンライン版）— 商品価値説明

## 一言サマリ
- 連想ワードだけで数字を並べる“協力推理”を、リアルタイム同期×観戦×ショータイム演出で再発明したブラウザゲーム。
- 「相談タイム」をドラッグ編集で可視化し、会話が最も盛り上がる瞬間を増幅。観戦だけでも“クスクス笑える”新しいソーシャル視聴体験。

## 体験価値（ユーザーに届く価値）
- 相談タイムの再発明: 失敗後の惰性を、ドラッグで並べ替え・編集できる“共同編集ボード”化。
- 一体感の演出: 参加者全員のカードを3D回転で一斉提示（Pixi×GSAPのショータイム）。
- “いつ行っても誰かいる”設計: メインメニュー＝ロビー一体UIで、今の熱量が即分かる。
- 一人でも楽しい観戦: 連想ワードにニヤリとできる“視聴専用モード”。参加せずとも満足できる。
- 快適な操作感: オンラインでも卓上にいるような軽さ（低遅延ドラッグ、音の立ち上がり）。

## コアアイデア（オリジナリティ）
- ドラッグ編集可能な“相談タイム”でコミュニケーションの楽しさを最大化。
- 背景テーマを切り替えるとショータイムが変化（演出・BGM・SFXが連動）。
- 観戦体験の階層化（無料ディレイ／低遅延チケット）で“観る楽しさ”を収益化に接続。
- ロビー常時可視×掲示板（開発者コメント→ニュース/Tips）で“常在感”を醸成。

## 想定ユースケース
- 友人・配信者の雑談コンテンツ／リモート飲み会のアイスブレイク。
- ボドゲ会の待ち時間に軽く1ゲーム。観戦しながら自然とルール学習。
- 一人時間のエンタメ（観戦チケットで“ながら視聴”）。

## 技術スタックとアーキテクチャ
- フロント: Next.js 14（App Router）/ TypeScript / Chakra UI / Pixi.js 8 / GSAP 3
- リアルタイム: Firebase（Firestore: 状態、RTDB: Presence 権威）/ Auth / Functions
- 状態管理: XState（FSM）— 複雑遷移を明示し UI 分岐を安定化
- PWA: Service Worker の安全更新（待機検知・自動/手動適用・BroadcastChannel・ループガード）
- 計測/可観測性: dumpItoMetrics() による perf/audio/drop 指標、トレースバッファ
- 決済: Stripe（観戦パス/テーマ販売に対応可能）

## 非機能要件とSLO（p95 目標）
- 観戦入室→盤面可視 ≤ 1.5s
- “今プレイ中”更新レイテンシ ≤ 0.4s
- ドラッグ反映（commit）≤ 50ms
- 音の立ち上がり ≤ 80ms
- SW 待機→適用完了 ≤ 5s（自動適用時）

## 計測設計（例）
- dumpItoMetrics() で取得できる主要キー
  - perf.*（warmup、prefetch/commit など）
  - audio.*（pointerToSoundMs、baseLatencyMs など）
  - client.drop.*（queueWaitMs）
  - lobby/ spectator.*（timeToHeat, joinToVisibleMs など）
- トレース: window.__ITO_TRACE_BUFFER__（直近の重要イベント）

## 差別化要因（置き換え困難性）
- “UI再現”では到達できない非機能の厚み: 安全更新、回線断復帰、タブ間連携、E2E/計測。
- 相談タイム×ショータイム×観戦の三点盛りで、体験価値に“堀（moat）”を形成。
- ロビー常時可視と掲示板で“常在感”を設計（定着率向上に直結）。

## 収益化モデル（例）
- 観戦チケット: 無料（数秒ディレイ）/ デイパス / 月額パス（低遅延・シアターモード・スタンプ・リプレイ枠）
- テーマ/ショータイムパック: 背景・カード絵・SFX/BGMの季節パック販売
- イベントパス: 期間限定ショー＋観戦特典
- 相互送客/アフィリエイト: ロビー/観戦画面からボドゲ商品へ誘導

## コミュニティ/成長戦略
- “今盛り上がっている部屋”の即可視化を最上段に固定→観戦→参加のCVを最大化。
- Discord連携で募集/合流を高速化。ロビーのトラフィックを外部コミュニティに循環。
- クリップ/名場面の共有（規約内）で二次拡散。

## 法務/権利の基本方針（要旨）
- 既存ボードゲームの名称・ロゴ・意匠は使用しない（混同回避）。
- 本作は独自名称・独自ブランドで運用。必要に応じてパートナーと連携（相互送客、コラボテーマ等）。
- 画像/音源/フォント/依存ライブラリのライセンス台帳を整備し、再配布可否・クレジット要否を明確化。

## KPI（初期目標の例）
- DAU / 観戦→参加CV（7–10%）/ 平均観戦15分 / 同時観戦者数
- ロビー初期可視 ≤ 750ms / “今プレイ中”更新 ≤ 0.4s / 観戦可視 ≤ 1.5s
- テーマ/チケットのARPPU、継続率（D30/D60）

## ロードマップ（90日イメージ）
- 0–30日: 独自ドメイン＋LP、無料観戦 公開、SLO計測の可視化
- 31–60日: 観戦チケット（低遅延/シアター/スタンプ）、ショータイム1stパック
- 61–90日: Discord募集連携、クリップ共有、相互送客PoC（提携先とKPI検証）

## 提携パターン（サマリ）
