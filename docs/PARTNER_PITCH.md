# Partner Pitch — 序の紋章 III（オンライン版）

**一言で**
- 連想ワードだけで数字を並べる協力推理ゲームを、リアルタイム×観戦×ショータイム演出で“いつ行っても誰かいる”体験に最適化。Webブラウザだけで成立し、見て楽しい・参加して楽しいを両立。

**体験価値（Why Us）**
- 相談タイムの再発明: 失敗後の“惰性の答え合わせ”を、ドラッグで気軽に並べ替える「会話が弾む編集時間」に変換。
- 観戦は一人でも楽しい: 観る専用導線（ディレイ有/無の二層）で“クスクス笑える”瞬間を量産。観戦→参加へのCVも設計。
- ショータイム演出: Pixi.js+GSAPで全員カードの3D提示。テーマ切替で見世物が変わる＝イベント/販売と相性抜群。
- “今ここで盛り上がっている”を可視化: ロビー常時可視×メインメニュー一体UI。入室直後に熱量が分かる。

**技術的な堅牢性（運用まで含めて）**
- リアルタイム同期: Firestore（状態）+RTDB（Presence）で観戦→再入室・回線断復帰を安全に。
- 安全なPWA更新: 待機検知/自動・手動適用/タブ間連携/ループガードで“事故らない更新”。
- 可観測性: dumpItoMetrics() ベースで perf/audio/drop を常時可視化。SLOをp95で管理。

**SLOベースライン（p95 目標）**
- 観戦入室→盤面可視 ≤ 1.5s
- “今プレイ中”更新レイテンシ ≤ 0.4s
- ドラッグ反映（commit）≤ 50ms
- 音の立ち上がり ≤ 80ms
- SW待機→適用完了 ≤ 5s

**60秒デモ台本（営業用）**
- 0–10s: ロビー入室→“今盛り上がっている部屋”が即見える
- 10–25s: 観戦で1.5s以内に盤面可視、リアクション
- 25–45s: 相談タイムでドラッグ編集→全員カード3Dショータイム
- 45–60s: 片タブで更新待機→自動適用→タブ間同期（安全更新）

**提携メニュー（選択式）**
- 共同イベント（非独占/短期）: コラボ・ショータイム1種＋物販リンク。KPI: 同時観戦者、商品CV。
- 公式デジタル・コンパニオン（年契約）: 学習/観戦/募集の公式動線。ロゴ・商標は相手基準。
- 相互送客/アフィリエイト: ロビー/観戦画面から商品へ誘導、UTMで計測。
- 限定独占（特定タイトル）: デジタル権の限定独占。期間・領域を限定して合意。

**観戦チケット（収益化案）**
- 無料観戦: 数秒ディレイ＋機能限定（十分楽しいが“物足りない”設計）
- 有料パス: 低ディレイ/シアターモード/スタンプ/全卓アクセス/リプレイ枠。日/月/イベント単位。

**PoC 提案（4–6週）**
- 目的: 相互送客と観戦価値の検証（DAU/同時観戦/商品CV）
- スコープ: コラボ・ショータイム1種、相互リンク、KPI計測基盤
- 成功基準（例）: 観戦→参加CV ≥ 7%、商品CTR ≥ 6%、同時観戦≥50

**置き換えコスト（参考）**
- 同等品質のゼロから再制作: 3,000–5,000万円（25–35人月×120–150万円）
- 価値の根拠: 運用まで含めた非機能（安全更新/復帰/計測）の厚み＋体験の堀

**次の一歩**
- 1) 30–60分のデモ＋Q&A
- 2) KPI合意（PoC）→ ショータイム1種/UTM設定
- 3) 4–6週で共同レポート→ 本提携の選択肢検討
