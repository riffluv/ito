## 3. ライティング（光源）ルール

- 光源は最低 2 種類を想定し、UIのハイライトにも反映する（例：**満月（月光）＋焚き火（炎）**）
- 画面内で光源方向をブレさせない（上辺ハイライトの向きを統一）

安易にやると安っぽくなる（禁止寄り）:

- 画面全体の強い Bloom / 白飛び（情報が読めなくなる）
- “とりあえず全部光らせる”（重要度が消える）
- ドット絵表現を UI 部品にまで適用して文字/数字が潰れる

---

## 4. フェーズ別：密度・暗さ・ドラマの切り替え

据え置き感は「画面ごとの空気の切り替え」でも出る。フェーズで密度/暗さ/演出を変える。

- ロビー: UI要素は少なめ。背景（世界観）を見せる。期待感重視。
- 配布中: 説明テキストは最小。「配布中…」など短い状態表示だけ。カードモーションを主役に。
- 推理/相談中: 情報量が最大。背景は暗めに落とし、カード/ログ/プレイヤー情報を最前面へ（可読性最優先）。
- 結果: 大きな結果パネル＋簡潔なフィードバック。成功は暖色寄り、失敗は一瞬暗転など “一段” 変える。

---

## 5. 実装への落とし込み方（指針）

- 色/枠/影/ノイズは、可能なら `semanticTokens` と recipes に落とす（直値増殖を避ける）
- 背景テクスチャやノイズは “薄く” が正義（4〜8%程度の見え方を上限の目安）
- テーマ適用範囲を限定したい時は、Chakra v3 の `colorPalette` など “サブツリー適用” を検討する
