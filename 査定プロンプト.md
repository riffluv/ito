あなたはゲーム開発の実務・経営・投資に精通したコンサルタントです。
以下の成果物を評価し、日本円ベースでの価値を多角的に算定してください。

対象:
- 個人開発で構築されたリアルタイム協力型カードゲーム（Next.js 14 / TypeScript / Chakra UI / Pixi.js / Firebase RTDB・Firestore・Auth・Functions / Stripe / PWA / Safe Update 機構 / XState FSM など）
- プロダクトはブラウザで完結、オンボーディング済み、アップデート自動適用フロー再構築済み
- リポジトリは非公開（企業はゼロから再現する前提）

評価観点:
1. **現在の資産価値（コードベース・アイデア・運用フロー）**  
   - 業務委託または企業が購入する場合の適正価格帯（円）  
   - 保守・運用ドキュメントの整備度合いを踏まえた評価  
   - PWA・リアルタイム同期・カードゲーム特有のUX価値

2. **再現コスト（企業が一から真似する場合）**  
   - プランニング／UI・UX／実装／テスト／運用準備のフェーズ別費用  
   - 想定チーム構成と工期、国内開発ベンダー水準での総額（円）  

3. **市場価値 / ライセンス価値**  
   - 類似サービス・競合プロダクトとの比較による差別化要因  
   - プレイフィールやアイデア面の独自性がどの程度価格に上乗せされるか  
   - 将来的なマネタイズポテンシャル（サブスク・課金・B2B利用など）

4. **技術的優位性・リスク**  
   - Safe Update や RTDB プレゼンス設計など、特に優れている箇所  
   - 技術的負債・依存リスク・保守上の懸念があれば定量化

5. **総合査定**  
   - 買収価格・共同開発契約・IPライセンス契約の3パターンで推奨価格帯（円）を提示  
   - 根拠となる数値・算出プロセスをわかりやすく説明  
   - 今後価値向上のために優先すべき改善提案

出力形式:
- 最初に結論サマリー（50〜100字程度、日本語）  
- その後、各観点を見出し付きで詳細記述  
- 金額はすべて日本円表記（カンマ区切り）で明示  
- 根拠に使った資料・推定値があれば簡潔に添える

補足:
- 与えた情報から推測可能な範囲で大胆に見積もって構いません。  
- 発言者の価値を矮小化せず、正当な評価を出してください。
